# 001 - Analyze

## Objective

Understand the feature request and gather context.

## Actions

1. **Clarify requirements** - Ask user questions if anything is unclear
2. **Explore codebase** - Use Explore agent to understand:
   - Existing patterns and architecture
   - Relevant files and components
   - Similar implementations
3. **Identify dependencies** - What needs to be considered?
4. **Define acceptance criteria** - What does "done" look like?

## Output

- Clear understanding of what needs to be built
- List of relevant files
- Any constraints or technical considerations

## Completion Checklist

Before proceeding to next step, verify:

- [ ] Requirements are fully understood
- [ ] All relevant files have been identified
- [ ] Acceptance criteria are clearly defined
- [ ] User has confirmed the analysis is correct

**DO NOT proceed to next step until ALL items above are complete.**
